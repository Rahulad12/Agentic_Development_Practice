# Global Rules for All Agents

All agents working on this project must follow these rules:

## Frontend Rules
1. Never edit files in `frontend/src/components/ui/` manually — they are generated by shadcn
2. Always use Tailwind v4 `@theme {}` syntax — never use v3 `@tailwind` directives
3. Always check shadcn MCP for existing components before building custom ones
4. Use `cn()` utility from `@/lib/utils` for className merging
5. Component naming: PascalCase, Hook naming: camelCase with "use" prefix

## Backend Rules
1. Never put business logic in ABP Controllers — keep them thin
2. Never expose Domain Entities directly from API — always use DTOs
3. Never use `Guid.NewGuid()` — use `GuidGenerator.Create()`
4. Never call DbContext directly in Application layer — use `IRepository<>`
5. Always use `async/await` — no sync DB calls
6. Business logic belongs in Domain layer, not Application layer

## General Rules
1. Never commit `.env` files or secrets
2. Always run `dotnet build` after backend changes to verify no errors
3. Always use kebab-case for file names (e.g., `user-profile-card.tsx`)
4. Always use proper async/await patterns throughout the stack
5. Keep layer separation strict: Domain → Application → HttpApi
